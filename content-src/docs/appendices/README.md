# appendices/

This folder holds **reference** notes -- templates, glossary, notation, and the library registry.

## Page structure (MDX body)

Reference notes have no strict heading template. Use whatever structure best serves the reference material. The `.mdx` file must **not** contain YAML frontmatter.

## JSON sidecar structure (*.meta.json)

**Required fields (build will fail without these):**

| Field | Type | Example |
|-------|------|---------|
| `title` | string | `"Glossary"` |
| `description` | string | `"Definitions of domain-specific terms."` |
| `id` | string | `"REF-GLOSS-0001"` (prefix `REF-`, then code, then number) |
| `type` | string | `"reference"` (must be exactly this) |

**Default values for references:**

References are terminal leaf nodes. They default to:
- `closure_score`: `6`
- `closure_status`: `"closed"`
- `review_cycle`: `"yearly"`

**Available connection fields:**

| Field | Links to |
|-------|----------|
| `related_notes` | Notes that use this reference |
| `related_standards` | Standards this supports |
| `tags`, `domain`, `status` | Metadata |

References do not require upward/lateral/applied links.

## Special files

- `library-registry/` -- Contains `index.mdx`, `index.meta.json`, and `library-registry.data.json`. Do not edit the JSON manually; it is generated by `tools/lib-registry/`.
- `templates/` -- Contains all nine note-type body and sidecar templates as on-site reference.

## Folder navigation (meta.json)

```json
{
  "title": "Appendices",
  "icon": "Book",
  "pages": ["templates", "glossary", "notation", "library-registry"]
}
```

## How to add a new reference

1. Create a new folder with the slug name under `appendices/`
2. Copy `_template/example.mdx.template` → `<slug>/index.mdx` and `_template/example.meta.json.template` → `<slug>/index.meta.json`
3. Fill in all sidecar fields
4. Write the `.mdx` body
5. Add the slug to `appendices/meta.json` `pages` array
6. Run `docker compose up --build` to rebuild and verify
