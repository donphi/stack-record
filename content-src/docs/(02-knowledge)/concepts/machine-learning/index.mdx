## What this map is

This map is the hub for machine-learning concepts that recur across projects: optimisation, generalisation, model families, and training dynamics at a level below any single codebase. It connects math and statistics to the verbs we use in pipelines (fit, score, calibrate). Start here when a doc names an algorithmic idea without pointing to a definition.

## Why this matters

Shared vocabulary keeps experiments comparable and reviews honest. When loss, regularisation, and optimisation are documented once and linked often, we avoid silent changes in what “trained model” means. It also makes onboarding faster for people crossing from classical software or domain science.

## Where this fits

Machine learning spans representation learning, retrieval methods, systems compute, and operational experiments. This index should stay thin; depth lives in child notes and in methods maps that describe how ideas are composed in practice.
